Week 12: Project Prep (MVP)

<br> <b> Overall project scope </b> This week I focused on the two most difficult components of my project: 1) designing a filter system with removable compartments and 2) Completing the code for app bluetooth connectivity to my water system. <u> Water filter </u> When thinking of feasible water filtration systems I could replicate, I came across this 5 stage filtration system on amazon. The system comprises of multiple layers, each of which will filter the water for different components based on the size and properties of the filling. The components are as follows: Activated carbon, ceramic balls, Resin, and two types of sand. I realized two potential improvements with this filtration system. <div class = "center"> <img src="../12/amazon.jpeg" width ="200" height="400"/> </div> Currently you can only buy packaged filter systems, however each commponent within this filter has a different shelf life. For example, activated carbon should be replaced every 6months- year while silica sand can last multiple years. In addition a lot of these materials can be sourced locally in rural communities, therefore communities may only need to buy certain parts such as the activated carbon. As a result my goal is to create a similar filter in which you can replace individual compartments allowing for a more cost-effective and sustainable system. <u> Fusion 360 </u> Here is my first prototype of the abovementioned system: <div class = "center"> <img src="../12/fusion.png" width ="600" height="400"/> </div> <iframe src="https://college418.autodesk360.com/shares/public/SH9285eQTcf875d3c539c97339a88419e3ed?mode=embed" width="640" height="480" allowfullscreen="true" webkitallowfullscreen="true" mozallowfullscreen="true" frameborder="0"></iframe> <u> Printing with PETG </u> I chose PETG because it is a known water-safe, water-proof filament that was available in the lab. I also had to adjust the z component on the printer to help the PETG stick better. <u>Atempt 1 prototype 1:</u> This print attempt was a complete failure (but failure = learning :) ). The goal behind this was to have a fully functioning app that would receive water level data from a sensor in the water system and also have two buttons to turn off and on the pump. I thought about programming it so that the pump would automatically shut off after a certain amount but I realized since this was a small scale water system it would make more sense to prioritize user autonomy so the user could turn off the pump whenever they didn't want to use it regardless of water level.
